## 4. 当前已实现功能

### 4.1 项目命令

已实现：

- `project create`
- `project list`
- `project show`
- `project status`

#### `project create`

功能：

- 创建项目目录结构
- 创建 `project.db`
- 创建 `project.yaml`
- 初始化 SQLite schema

项目目录结构示例：

```text
workspace/projects/<project_slug>/
  project.yaml
  project.db
  shots/
  outputs/
  logs/
  previews/
```

#### `project status`

功能：

- 汇总项目下 shot 数量
- 按 `pipeline_status` 统计镜头数量

当前输出类似：

- `project: demo-project`
- `total_shots: 2`
- `draft: 2`

### 4.2 分镜命令

已实现：

- `shot add`
- `shot list`
- `shot show`
- `shot status`
- `shot import-video`
- `shot scan-imports`

#### `shot add`

功能：

- 创建 shot 记录
- 初始化分镜目录：
  - `imports/`
  - `generated/`
  - `normalized/`
  - `audio/`
  - `subtitles/`
  - `previews/`

#### `shot import-video`

功能：

- 校验项目存在
- 校验 shot 在数据库中真实存在
- 复制原始输入到 `imports/`
- 调用 FFmpeg 标准化到 `normalized/clip.mp4`
- 写入 `media_assets`
- 将 shot 状态推进到 `video_ready`

#### `shot scan-imports`

功能：

- 扫描 `shots/*/imports/`
- 支持扩展名：
  - `.mp4`
  - `.mov`
  - `.mkv`
  - `.webm`
- 缺少 `imports/` 目录时会安全跳过，不会崩溃
- 若 shot 目录存在但数据库里无该 shot，会报错阻断，避免脏数据
- 成功后走与手动导入一致的标准化与状态推进逻辑

#### `shot status`

功能：

- 输出单镜头当前状态
- 输出关键资产存在性：
  - `has_normalized_video`
  - `has_audio`
  - `has_subtitles`

### 4.3 预览功能

已实现：

- `preview build`

功能：

- 生成本地 HTML 预览页：
  - `workspace/projects/<slug>/previews/index.html`
- 展示：
  - 项目基础信息
  - shot 列表
  - prompt/script
  - 当前状态
  - 视频相对路径
  - `<video>` 预览

补充说明：

- 预览模板 `index.html.j2` 已显式加入 wheel 打包规则
- 已验证 wheel 中包含模板文件

### 4.4 音频功能

已实现：

- `audio generate`

功能：

- 针对 `video_ready` 的 shot 生成音频
- 当前默认 provider 为 `fake`
- 输出：
  - `audio/voice.wav`
- 写 `voice_audio` 资产
- 将 shot 状态推进到 `audio_ready`

### 4.5 TTS Provider 骨架

已实现：

- `TTSProvider` 协议
- `FakeTTSProvider`
- 环境变量与 CLI 参数切换 provider

当前配置逻辑：

- 默认 provider：`fake`
- 环境变量：`ANIME_PIPELINE_TTS_PROVIDER`
- CLI 参数：`audio generate --provider <name>`

当前只实现了：

- `fake`

当前 `fake` provider 的行为：

- 生成一个占位 WAV 文件
- 按文本长度粗略决定静音帧数

未实现但建议下一步可做：

- 本地 TTS Provider
- 云端 TTS Provider

### 4.6 字幕功能

已实现：

- `subtitle generate`

功能：

- 处理 `audio_ready` 的 shot
- 生成 `subtitles/subtitles.srt`
- 写 `subtitle_file` 资产
- 将状态推进到 `subtitle_ready`

当前字幕逻辑：

- 按中英文标点切句
- 保留标点
- 按镜头 `target_duration` 分配总时长
- 按 cue 去空白、去标点后的文本长度做加权分配
- 最小总时长兜底为 1 秒

当前支持切句的标点包括：

- `。`
- `！`
- `？`
- `!`
- `?`
- `；`
- `;`
- `，`
- `,`

### 4.7 导出功能

已实现：

- `export rough-cut`
- `export final`

#### `export rough-cut`

功能：

- 拼接项目中存在 `normalized/clip.mp4` 的镜头
- 生成：
  - `outputs/rough_cut.mp4`

注意：

- 当前 rough-cut 以“存在标准化视频”为准，不依赖单一状态值
- 这样即便 shot 已进入 `audio_ready` 也仍可参与 rough-cut

#### `export final`

当前实现是：

- 视频 + 音频 + 软字幕流封装

不是硬烧录字幕。

当前逻辑：

1. 只筛选 `pipeline_status == subtitle_ready` 的镜头
2. 对每个可导出镜头要求以下资源必须完整：
   - `normalized/clip.mp4`
   - `audio/voice.wav`
   - `subtitles/subtitles.srt`
3. 若任一关键资源缺失，直接失败，并汇总缺失项
4. 每个镜头先生成一个中间文件：
   - `outputs/<shot_no>_subtitled.mp4`
5. 中间文件中：
   - 视频来自 `normalized/clip.mp4`
   - 音频优先来自 `voice.wav`
   - 字幕封装为 `mov_text`
6. 最后 concat 为：
   - `outputs/final.mp4`

#### Final 导出的关键设计

- 不会因为 `draft` shot 而阻塞 final
- 只对 `subtitle_ready` 的镜头做完整性校验
- 关键资源缺失时会报清晰错误，而不是静默跳过
- 最终文件当前应包含：
  - 音频流
  - 字幕流

### 5.2 当前实际使用到的状态

当前主线实际用到并经过测试覆盖的状态包括：

- `draft`
- `video_ready`
- `audio_ready`
- `subtitle_ready`

语义：

- `draft`
  - 新建镜头，尚未导入/标准化视频
- `video_ready`
  - 已有标准化视频
- `audio_ready`
  - 已生成 `voice.wav`
- `subtitle_ready`
  - 已生成 `.srt`，可进入 final export

## 8. 当前已知限制

### 8.1 音频仍是 fake provider

虽然已经有 TTS Provider 骨架，但真实 TTS 还没接。

当前 `voice.wav` 仍然是占位音频，不是真实语音。

### 8.2 字幕时间分配仍然是基础增强版

目前只是：

- 按标点切句
- 按总时长分配
- 按文本长度加权分配 cue 时长

还没有：

- 基于音频长度/语速的更真实分配
- forced alignment
- 词级/字级字幕

### 8.3 Final 是软字幕，不是硬烧录

当前 `final.mp4` 采用的是：

- 音频流
- `mov_text` 字幕流

不是硬烧录到画面上的字幕。

### 8.4 媒体标准化参数目前写死

当前标准化参数是固定的：

- `160x90`
- `30fps`
- `mpeg4`
- `aac`

还没有做成配置驱动。
